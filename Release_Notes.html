Overview

  

•  The X-CUBE-CLD-GEN (STM32 generic IoT software expansion for STM32Cube) provides ready-to-run firmware examples to support quick evaluation and development of MQTT and HTTPS IoT Cloud applications on B-L475E-IOT01, 32F413H-DISCO, 32F769I-DISCO and 32L496G-Discovery with cellular add-on boards.

•  3 IDEs are supported: IAR EWARM, Keil MDK-ARM and System Workbench for STM32.

•  It contains BSP compatible Sensor drivers for B-L475E-IOT01.

•  X-CUBE-CLD-GEN runs on five platforms.

º  The B-L475E-IOT01A and 32F413HDISCOVERY support Wi-Fi® connectivity with an on-board Inventek ISM43362 module.

º  The 32F769IDISCOVERY board provides a native Ethernet interface.

º  The P-L496G-CELL01 and P-L496G-CELL02 packs, with the 2G/3G Quectel UG96 and LTE Quectel BG96 cellular modem daughterboards respectively, support cellular connectivity.

•  Below the most useful document:

º  UM2347 - STM32 Generic IoT software expansion for STM32Cube.

Known Limitations

•  Network connection instability on B-L475E-IOT01 and STM32F413H-DISCO:

º  The board may not connect to the WiFi access point at first trial.

º  The resolution of the remote host name sometimes fails - it seems to depend on the cache status of the DNS server of the local network.

º  Sporadic AT protocol errors on the serial link between the STM32 host and the WiFi module (control/data unsync).

–  -> In any of those cases, a board reset is required.

º  Remote server disconnection reported as a system error by the WiFi module.

•  System Workbench for STM32.

º  B-L475E-IOT01 board rev B and rev C requires to use System Workbench with version greater or equal to 2.1. Debug option must be changed.

–  The "reset mode" option must be set to "software system reset". This option is visible when clicking on the "show generator options" from the debug tab.

–  The selected configuration must be the "Ac6 STM32 debug config".
